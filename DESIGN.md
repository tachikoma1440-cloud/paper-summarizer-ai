# Academic Slide Design Guidelines

このドキュメントは、本プロジェクトで生成するすべての発表用スライド（抄読会資料・進捗報告）における視覚的デザインを規定します。
`backend/main.py` の PPTX 生成コードはこの仕様に準拠して実装・変更してください。

---

## 1. カラーパレット (Color Palette)

| 役割 | 名前 | HEX | 用途 |
|------|------|-----|------|
| Base | White | `#FFFFFF` | 通常スライドの背景・本文テキスト |
| Primary | Deep Navy | `#003366` | 通常スライドの見出し・重要テキスト。表紙と中扉は背景色として使用 |
| Secondary | Sky Blue | `#0EA5E9` | グラフ要素、見出しの背景座布団 |
| Accent | White | `#FFFFFF` | 濃紺背景上の文字色 |
| Alert | Red / Rose | `#EF4444` | 課題・エラー・注意点の強調（**太字**のみ。縁取り禁止） |
| Highlight | Royal Blue | `#2563EB` | 数値・ロジックの強調 |

### コード対応（`backend/main.py`）

```python
_WHITE   = RGBColor(0xFF, 0xFF, 0xFF)   # Base
_NAVY    = RGBColor(0x00, 0x33, 0x66)   # Primary  (#003366)
_SKYBLUE = RGBColor(0x0E, 0xA5, 0xE9)   # Secondary (#0EA5E9)
_ALERT   = RGBColor(0xEF, 0x44, 0x44)   # Alert    (#EF4444)
_ROYAL   = RGBColor(0x25, 0x63, 0xEB)   # Highlight (#2563EB)
_BLACK   = RGBColor(0x00, 0x00, 0x00)   # 本文（補助）
```

> **注意**: `_NAVY` の現在の実装値は `#1F497D` です。新規スライドを追加する際は必ず `#003366` を使用してください。既存スライドのリファクタリング時にも合わせて修正すること。

---

## 2. タイポグラフィ (Typography)

- **日本語フォント**: `Noto Sans JP`（和文はゴシック体で可読性を重視）
- **英数字フォント**: `Arial`（ユニバーサルデザインで標準的なフォント）

### フォントサイズ基準

| 用途 | サイズ | スタイル |
|------|--------|---------|
| スライドタイトル（メッセージ行） | 36–44 pt | **Bold** |
| セクション見出し | 28–32 pt | Bold |
| 本文テキスト | 20–24 pt | Regular |
| 図表内の文字・グラフ軸・単位 | 最低 18 pt | Regular |
| 下部注記・文献引用 | 16 pt | Regular |

> `python-pptx` での指定例: `run.font.size = Pt(22)` / `run.font.name = "Arial"`

---

## 3. 全体レイアウトと構造 (Layout & Structure)

### マージン（余白）
スライドの四辺（上下左右）に必ず **2 cm 以上**のセーフティマージンを確保する。

```python
# Inches(0.79) ≈ 2 cm
MARGIN = Inches(0.79)
```

### 画面の面積構成（スライド高さを基準）

```
┌──────────────────────────────────────────┐
│  上部 25%：スライドタイトル（メッセージ）   │  ← 結論・主張を1文で
├──────────────────────────────────────────┤
│                                          │
│  中央 65%：主要内容                       │  ← 図・グラフ・要点
│  （文字の羅列を避け One Slide, One Message）│
│                                          │
├──────────────────────────────────────────┤
│  下部 10%：補足情報・引用文献             │  ← 著者名, 雑誌名, 年号
└──────────────────────────────────────────┘
```

| 領域 | 高さの比率 | 内容 |
|------|-----------|------|
| 上部 | 25% | スライドタイトル。「このスライドで言いたい結論」を記載（項目名 NG） |
| 中央 | 65% | 主要内容。図・グラフ・要点を中心に配置 |
| 下部 | 10% | 引用文献（著者名, 雑誌名, 年号）や注記 |

### 表紙・中扉スライド

- 背景全面を `#003366`（Deep Navy）で塗りつぶす
- タイトル・サブタイトルは `#FFFFFF`（White）で表示
- 水平区切り線を `#0EA5E9`（Sky Blue）または `#FFFFFF` で引く

---

## 4. 発表用の特別ルール

1. **データソースの明記**: グラフや表に出典がある場合は下部 10% 領域に省略せず記載すること。
   - 例: `Doe J, et al. N Engl J Med. 2024;390:123-134.`
2. **コントラスト比**: 会場の後方席からプロジェクター越しに見える配色を維持する。白背景に薄い色のテキストは禁止。
3. **One Slide, One Message**: 中央領域に詰め込みすぎず、1枚のスライドで伝えるメッセージは1つに絞る。
4. **Alert 色の使い方**: `#EF4444` は**太字テキストのみ**で使用し、図形の背景色や縁取りには使わない。
5. **Highlight 色の使い方**: `#2563EB` は数値・統計値・重要な論理ステップのみに適用し、装飾目的には使わない。

---

## 5. Obsidian 連携ノートとの関係

スライド生成に使用するサマリーテキストには Obsidian 内部リンク `[[用語]]` が含まれる場合があります。
PPTX 生成時は `_strip_md()` 内で自動的に除去されます（`[[心筋梗塞]]` → `心筋梗塞`）。
Markdown ダウンロード・Obsidian 保存時にはリンクがそのまま残ります。
